Created to get information out of ledger to pull into and compare with data in external company’s back office


What is a SynApps Plugin?

  • Pluggable way to extend SynAPPS functionality without adding custom functionality
  • Examples
    • Doing back office validation of proposed contracts
    • Warehousing data
    • Aggregating information to create a new ‘SA Event’
    • Tying SynAPPS to an accounting system

Neo4J Plugin

Solves problem of integrating into Neo4J Database and updates SynDNA Server


Kafka

Types:

  • Apache Kafka
  • Confluent Kafka

Apache Kafka (used in EBX)


  1. Publish events to Kafka to append to log
  2. Many can publish to a log
  3. Many can read from log
  4. Can categorize by topics (can publish events specific to a topic)
  5. Every consumer knows where they left off

In context of EBX:

  • Every event in EBX is posted to Kafka log
  • Plugins listen for events and read new events (whatever they choose to do after is their scope)
  • Idea is to create its own topics over time (dynamically)

How It Works


  • Plugin Registration (when you launch plugin it logs in through password, receives token, plugin is authenticated)
  • Service publishes notifications to a specific Kafka Topic
    • Can publish to multiple Topics simultaneously, but EBX currently only uses 1 topic for ease of use
  • Plugin consumes notifications form the Kafka Topic

PluginAPI


def plugins(serverUri: String): Future[Set[RegisteredPlugin]]
  • Returns a list of plugins that have been registered 

def registerPlugin(serverUri: String, p: PluginRegistrationRequest): Future[PluginRegistrationResponse]
  • Registers a new plugin and returns information required by the plugin to connect to Kafka 

def unregisterPlugin(serverUri: String, p: PluginUnregistrationRequest): Future[StatusCode]
  • Unregisters an already registered plugin when the plugin stops running 


def refreshPlugin(serverUri: String, p: PluginTokenRefreshRequest): Future[PluginTokenRefreshResponse]
  • Since the token returned by ‘registerPlugin’ only lasts for a day, the plugin needs to refresh its token every so often
